Colour Processing Study Designing an accurate colourimetric processing technique for an eHealth Microplate Reader application

Aims

The aim of this research was to improve the accuracy of the colour processing module of a prototype smartphone microplate reader application developed by Bellairs et. al. 1 The module was required to acheive an accuracy of within 5% of the commercial microplate reader gold standard.

Design

Well Detection Module

The well detection module had previously been designed to detect circles in the image live, while the user was taking a picture of a plate. This caused performance issues as each frame needed to be processed. The new well detection module has been designed to only processes the image after it has been taken. This was done to mititgate the performance problems and therefore enhance the application. The core detection algorithm remained unchanged.


Colour Processing Module

The colourimetric module is designed to have two seperate implementations so they may be compared and the most accurate be used in the final application. Both implementations convert the colour of a well into a concentration estimate. The first implementation converts the RGB pixel values to the CIELAB colour space 2 and this was used to calculate a saturation value for the well which is used to estime the concentration. The second implementation converts the RGB pixel values into a single greyscale intensity value for each well which is used to estimate the concentration. The principle concept behind both of the implementations is that the actual colour of the well is irrelevant. It is the saturation or intensity of the colour which indicates a high or low concentration.


Architecture Diagram

Implementation

Well Detection Module

The well detection is implemented with OpenCV v3.1, an Open Source image processing library3. The image is first converted to greyscale to simplify the processing by reducing the image to a single intensity channel. OpenCV then applies a Canny Edge Detector to detect potential edge points in an image. OpenCV then applies the Hough Circle Transform (HCT) method to detect circles. To simplify the colour analysis module, we replace each circle’s radius with the average radius over all the detected circles.


Colour Processing Module

Both implementations receive the well locations and the average circle radius from the well detection module. A square area covering the centre of each well is used to calculate the average RGB values for each well.


CIELAB Implementation

The RGB values are converted to the CIE1931 colour space and from there they are converted to the CIELAB colour space. This is because RGB values cannot be directly converted to the CIELAB colour space. A saturation value is then calculated and used along with concentration values to define a standard curve for a row of a specific compound. This standard curve can then be used to estime the concentrations for other wells of the same compund by substituting in their saturation value.


RGB Intensity Implementation

The RGB values are converted into a single intensity value for each well. This is used to define a standard curve for a row of a specific compound. This standard curve can then be used to estime the concentrations for other wells of the same compund by substituting in their intensity value.

Results

The results from this research showed that the CIELAB implementation was a lot more accurate in every test than the intensity implmentation. The CIELAB implementation also proved to be more accurate than the prototype application in some tests where the prototype was more accurate in others. Overall the application acheived an average percentage error of 9,3% which is worse than the 7% acheived by the prototype. Future research could investigate a hybrid approach between the two applications to use whichever approach is more accurate in each scenario.
